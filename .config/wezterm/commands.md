# WezTerm コマンド一覧

- **Leader**: `Ctrl+b`。押したあと **いったんキーを離してから** 2 秒以内に次のキーを押す（例: `w` でワークスペース一覧）
- **SUPER**: macOS では `Cmd` キー

---

## 根本原因: 設定ファイルが読まれていない

**ワークスペースの名前変更・削除や、キーバインドが一切効かない場合**は、WezTerm が **dotfiles の設定（`~/.config/wezterm/wezterm.lua`）を読んでおらず、別の設定かデフォルトで動いている可能性が高いです。

### なぜそうなるか

1. **`~/.wezterm.lua` が存在する**  
   WezTerm は環境変数や `--config-file` がなければ、`$HOME/.config/wezterm/wezterm.lua` を優先して使いますが、バージョンや環境によっては `~/.wezterm.lua` が先に使われることがあります。  
   また、別のツール（例: Claude の terminal-setup など）が `~/.wezterm.lua` を自動作成している場合があります。
2. **`~/.config/wezterm/` が存在しない**  
   home-manager をまだ `home-manager switch` していない、または別マシンで dotfiles をデプロイしていないと、このディレクトリがなく、WezTerm は `~/.wezterm.lua` にフォールバックします。
3. **設定の読み込みに失敗している**  
   `wezterm.lua` や `keybinds.lua` のエラーで読み込みが失敗すると、WezTerm はデフォルト設定で起動し、カスタムキーやワークスペース操作が効きません。

### 確認手順

1. **どの設定を読んでいるか確認する**  
   WezTerm を開き、**Ctrl+Shift+L** でデバッグオーバーレイを開く → 入力欄に `wezterm.config_file` と打つと、実際に読み込んでいる設定ファイルのパスが表示されます。
2. **`~/.wezterm.lua` の有無を確認する**  
   ターミナルで `ls -la ~/.wezterm.lua`。存在する場合は、中身が dotfiles の設定を読む内容か、あるいは削除/リネームしてよいか判断してください。
3. **`~/.config/wezterm/wezterm.lua` の有無を確認する**  
   `ls -la ~/.config/wezterm/wezterm.lua`。home-manager でデプロイしているなら、ここに dotfiles の設定がある想定です。

### 対処: dotfiles の設定を必ず読ませる

**`~/.wezterm.lua` が存在する、または「どの設定を読んでいるか」が `~/.config/wezterm/wezterm.lua` になっていない場合**は、次のどちらかを行ってください。

- **A. `~/.wezterm.lua` を dotfiles の設定を読むだけのファイルにする**  
  dotfiles に `wezterm-home.lua` を用意しています。これを `~/.wezterm.lua` にコピーまたはシンボリックリンクしてください。  
  ```bash
  ln -sf ~/.config/wezterm/wezterm-home.lua ~/.wezterm.lua
  ```  
  （home-manager で `~/.config/wezterm` を管理している場合。パスは環境に合わせて読み替えてください。）
- **B. `~/.wezterm.lua` を消す**  
  `~/.wezterm.lua` を削除またはリネームし、WezTerm が `~/.config/wezterm/wezterm.lua` を読むようにします。  
  ```bash
  mv ~/.wezterm.lua ~/.wezterm.lua.bak
  ```

いずれかの後、WezTerm を**完全に終了してから**再起動し、もう一度 **Ctrl+Shift+L** → `wezterm.config_file` で、意図した設定ファイルが読まれているか確認してください。

### ワークスペース（Ctrl/Leader が効かない環境向け）

- **ワークスペース一覧（ランチャーUI）**: **Cmd+Option+W**（Cmd+Alt+W）で開く。  
  （Cmd+Shift+W は macOS の「ウィンドウを閉じる」で使われることが多く、WezTerm に届かない）
- **作成・切り替え**: **Cmd+p** でコマンドパレットを開き、「workspace」などで検索して作成・切り替え用のコマンドを選ぶ。一覧表示のコマンドはパレットには出てこない。

### キーが効かないときのデバッグ（home-manager のみ利用の場合）

1. WezTerm を**既に開いた状態**で、その中で新規タブを開く。
2. そのタブで `which wezterm` または `command -v wezterm` を実行し、wezterm のパスを確認する。
3. `wezterm.lua` で `config.debug_key_events = true` にし、**Cursor の統合ターミナル**など別のターミナルで、手順2で出たパス（例: `~/.local/state/home-manager/profile/bin/wezterm`）を実行して WezTerm を起動する。
4. 出た WezTerm でキーを押すと、コマンドを実行したターミナルにキーイベントのログが出る。確認後は `debug_key_events = false` に戻す。

## ワークスペース

| キー | 操作 |
|------|------|
| `Cmd` + `Option` + `W` | ワークスペース一覧（ランチャー）を表示 |
| `Cmd` + `Shift` + `W` | 同上（OS に奪われない環境なら） |
| `Leader` + `w` | ワークスペース一覧を表示して切り替え |
| `Leader` + `$` | 現在のワークスペース名を変更 |
| `Leader` + `Shift` + `W` | 新規ワークスペースを作成して切り替え |
| **Cmd+p** | コマンドパレットからワークスペースの作成・切り替え（一覧UIは出ない） |

---

## コマンドパレット

| キー | 操作 |
|------|------|
| `Cmd` + `p` | コマンドパレットを開く |
| `Ctrl` + `Shift` + `p` | コマンドパレットを開く |

---

## タブ

| キー | 操作 |
|------|------|
| `Ctrl` + `Tab` | 次のタブへ |
| `Ctrl` + `Shift` + `Tab` | 前のタブへ |
| `Leader` + `{` | タブを左に移動 |
| `Leader` + `}` | タブを右に移動 |
| `Cmd` + `t` | 新しいタブを開く |
| `Cmd` + `w` | 現在のタブを閉じる |
| `Cmd` + `1` 〜 `8` | 1〜8番目のタブに切り替え |
| `Cmd` + `9` | 最後のタブに切り替え |

---

## ペイン（分割）

| キー | 操作 |
|------|------|
| `Leader` + `d` | ペインを縦に分割 |
| `Leader` + `r` | ペインを横に分割 |
| `Leader` + `x` | 現在のペインを閉じる |
| `Leader` + `h` | 左のペインにフォーカス |
| `Leader` + `l` | 右のペインにフォーカス |
| `Leader` + `k` | 上のペインにフォーカス |
| `Leader` + `j` | 下のペインにフォーカス |
| `Ctrl` + `Shift` + `[` | ペイン選択モード（矢印で選択） |
| `Leader` + `z` | 現在のペインをズーム（最大化/元に戻す） |

---

## ペインリサイズ（キーテーブル: resize_pane）

| キー | 操作 |
|------|------|
| `Leader` + `s` | リサイズモードに入る |
| `h` / `l` / `k` / `j` | 左 / 右 / 上 / 下にリサイズ |
| `Enter` | リサイズモードを終了 |

---

## ペイン切り替え（キーテーブル: activate_pane）

| キー | 操作 |
|------|------|
| `Leader` + `a` | ペイン切り替えモードに入る（1秒間有効） |
| `h` / `l` / `k` / `j` | 左 / 右 / 上 / 下のペインにフォーカス |

---

## コピー・貼り付け

| キー | 操作 |
|------|------|
| `Leader` + `[` | コピーモードを開始 |
| `Cmd` + `c` | 選択範囲をクリップボードにコピー |
| `Cmd` + `v` | クリップボードから貼り付け |

---

## コピーモード内の操作

### 移動

| キー | 操作 |
|------|------|
| `h` / `j` / `k` / `l` | 左 / 下 / 上 / 右 |
| `0` | 行の左端へ |
| `^` | 行の先頭（空白除く）へ |
| `$` | 行の末尾へ |
| `w` / `b` / `e` | 単語単位で前/後/末尾へ |
| `g` | スクロールバッファの先頭へ |
| `G` | スクロールバッファの末尾へ |
| `H` / `M` / `L` | 画面上端 / 中央 / 下端へ |
| `t` / `f` | 次の文字まで（t: 含む手前、f: 含む） |
| `T` / `F` | 前の文字まで |

### スクロール

| キー | 操作 |
|------|------|
| `Ctrl` + `b` / `f` | ページアップ / ダウン |
| `Ctrl` + `d` / `u` | 半ページ下 / 上 |

### 選択

| キー | 操作 |
|------|------|
| `v` | セル選択モード |
| `Ctrl` + `v` | ブロック選択モード |
| `V` | 行選択モード |

### コピー・終了

| キー | 操作 |
|------|------|
| `y` | 選択範囲をクリップボードにコピー |
| `Enter` | コピーしてコピーモード終了 |
| `Escape` / `Ctrl` + `c` / `q` | コピーモード終了 |
